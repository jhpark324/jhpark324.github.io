## Git 명령어



##### git init

`add init`

새로운 git 저장소가 만들어지며 해당 폴더를 git으로 관리가 가능

---

##### git add

`add <파일명>`

working directory에서 staging area로 옮기며 <파일명> 혹은 . 을 통해 선택함 

---

##### git commit

`git commit -m "설명"`

staging area에 있는 파일들을 하나의 버전으로 repository로 옮김

---

##### tracked

working directory에 있는 파일이 staging area에 있으면 해당 파일은 **tracked** 상태

---

##### untracked

working directory에 있는 파일이 staging area에 없으면 해당 파일은 **untracked** 상태

---

##### git log

repository에 commit 로그 확인. (아이디 조회후 git checkout과 연계)

---

##### git checkout

`git checkout <commit_id>`

repository에 커밋된 여러 버전에서 특정 버전을 선택해 working directory의 tracked 영역과 staging area에 붙여넣음 (HEAD가 가르키는 버전)

`git checkout HEAD^`  / `git checkout HEAD~1` 
^(캐럿) 갯수 / ~n 만큼 부모 commit으로 이동
이동원리 : 각 버전들은 이전 버전의 정보를 가지고 있음

---

##### git status

`git status`

- staging area와 최신 커밋의 차이점
- tracked 상태와 staging area 간의 차이점
- untracked 상태에 있는 파일

등의 정보를 보여줌.

---

##### git diff

`git diff`

staging area 와 tracked간의 차이를 자세하게 보여줌

`git diff <commit_id>`

tracked와 커밋간의 차이를 자세하게 보여줌

`git diff --cached`

커밋과 staging area의 차이를 자세하게 보여줌

`git diff <commit_id_1> <commit_id_2>`

커밋간의 차이를 자세하게 보여줌
