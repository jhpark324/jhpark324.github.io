## Git 명령어



##### git init

`add init`

새로운 git 저장소가 만들어지며 해당 폴더를 git으로 관리가 가능

---

##### git add

`add <파일명>`

working directory에서 staging area로 옮기며 <파일명> 혹은 . 을 통해 선택함 

---

##### git commit

`git commit -m "설명"`

staging area에 있는 파일들을 하나의 버전으로 repository로 옮김

`git commit -am "설명"`

add (tracked 파일) + commit을 동시에 수행 (수정, 삭제된 파일만)

---

##### tracked

working directory에 있는 파일이 staging area에 있으면 해당 파일은 **tracked** 상태

---

##### untracked

working directory에 있는 파일이 staging area에 없으면 해당 파일은 **untracked** 상태

---

##### git log

repository에 commit 로그 확인. (아이디 조회후 git checkout과 연계)

---

##### git checkout

`git checkout <commit_id>`

repository에 커밋된 여러 버전에서 특정 버전을 선택해 working directory의 tracked 영역과 staging area에 붙여넣음 (HEAD가 가르키는 버전)

`git checkout HEAD^`  / `git checkout HEAD~1` 
^(캐럿) 갯수 / ~n 만큼 부모 commit으로 이동
이동원리 : 각 버전들은 이전 버전의 정보를 가지고 있음

---

##### git status

`git status`

- staging area와 최신 커밋의 차이점
- tracked 상태와 staging area 간의 차이점
- untracked 상태에 있는 파일

등의 정보를 보여줌.

---

##### git diff

`git diff`

staging area 와 tracked간의 차이를 자세하게 보여줌

`git diff <commit_id>`

tracked와 커밋간의 차이를 자세하게 보여줌

`git diff --cached`

커밋과 staging area의 차이를 자세하게 보여줌

`git diff <commit_id_1> <commit_id_2>`

커밋간의 차이를 자세하게 보여줌

---

##### git branch

- 브랜치를 통해 독립적인 공간에서 코드를 실험적으로 짤 수 있음
- 다른 브랜치에서 개발한 내용을 명령어로 가져올 수 있음
- 브랜치를 많이 만들어도 폴더 복사의 개념이 아니라 참조의 개념이라 용량을 많이 차지하지 않음.

`git branch`

브랜치의 목록을 보여줌 

`git branch <branch_name>`

head가 가르키고 있는 커밋의 범위의 브랜치 생성

`git checkout <branch_name>`

브랜치 이동

`git switch <branch_name>`

브랜치 이동

| 주의사항: 다른 브랜치로 이동하는 경우 워킹 디렉토리를 클린하게 하자

---

##### git merge

`git merge <branch>`

특정 branch를 헤드가 가르키고 있는 곳에 merge<br>

머지커밋은 머지된 브랜치들을 부모로 둠

- 3-way merge

  - main branch에도 새로운 커밋, 다른 branch에도 새로운 커밋들이 존재한채로 merge

  - conflict(merge중에 diff충돌) 발생

- fast-forward merge

  - 다른 branch에서만 새로운 커밋 존재한채로 merge
  - main브랜치의 범위가 머지한 브랜치의 범위와 같아짐
  - 커밋 히스토리가 단순함

  | 주의사항: 워킹 디렉토리가 클린할때만 머지하자
  
   

---

##### git cherry-pick

`git cherry-pick <commit_id>`

- 특정 커밋만 가져옴

- conflict 오류 발생 가능
  - 발생시 선택 후에  `git cherry-pick --continue` 로 진행
